# RAG tutorial

An interactive, runnable tutorial that builds Retrieval-Augmented Generation
from first principles. Each section is a Jupyter notebook in `tutorial/`,
generated by a Python script in `scripts/`.

## Layout

```
.
├── README.md                 # this file
├── requirements.txt          # pinned-floor dependency list
├── key.env                   # your Gemini API key, one line
├── tutorial/                 # the executable notebooks (you read these)
│   ├── 01_retrieval_basics.ipynb
│   ├── 02_semantic_retrieval.ipynb
│   ├── 03_basic_rag.ipynb
│   ├── 04_embedding_models.ipynb
│   ├── 05_reranking.ipynb
│   ├── 06_evaluation.ipynb
│   ├── 07_advanced_rag.ipynb
│   └── 08_failure_cases.ipynb
└── scripts/                  # generator scripts that emit the .ipynb files
    ├── build_01_retrieval_basics.py
    ├── build_02_semantic_retrieval.py
    └── ...
```

## Prerequisites

- **Python 3.10+** (developed and tested on Python 3.14 on Windows).
- ~3 GB of free disk for the HuggingFace model and dataset caches.
- An internet connection on first run (HuggingFace downloads + Gemini API calls).
- A **Google Gemini API key**. Get one for free at
  <https://aistudio.google.com/app/apikey>.

## Setup

### 1. Get the Gemini key

Create `key.env` in the project root, containing **just** the key (no quotes,
no `KEY=` prefix, no newline at the end is required):

```
AIzaSy....your-actual-key....
```

The notebooks load it via:

```python
import os
from pathlib import Path
os.environ["GEMINI_API_KEY"] = (Path("..") / "key.env").read_text().strip()
```

### 2. Create the virtual environment and install dependencies

#### Windows (PowerShell)

```powershell
py -3 -m venv venv
.\venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

If `Activate.ps1` is blocked by execution policy, run once per terminal:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned
```

#### macOS / Linux (bash / zsh)

```bash
python3 -m venv venv
source venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

### 3. (Optional but recommended) Register the venv as a Jupyter kernel

```powershell
python -m ipykernel install --user --name rag-tutorial --display-name "RAG tutorial (venv)"
```

When opening a notebook, pick **RAG tutorial (venv)** as the kernel.

## Running the notebooks

Start Jupyter from the repo root **with the venv active** so the kernel sees
the installed packages:

```powershell
jupyter notebook tutorial/
```

Or open the folder in VS Code and pick the venv's interpreter (the
"Python: Select Interpreter" command will list `venv\Scripts\python.exe`).

Work through the notebooks in numerical order — each builds on the previous.

## What gets downloaded on first run

The notebooks auto-fetch the following from the HuggingFace Hub the first
time they run. After that everything is cached under
`~/.cache/huggingface/` and runs offline.

### Models

| model                                                | size    | used in   |
|------------------------------------------------------|---------|-----------|
| `sentence-transformers/all-MiniLM-L6-v2`             | ~80 MB  | nb 2, 3, 5, 6, 7, 8 |
| `cross-encoder/ms-marco-MiniLM-L-6-v2`               | ~80 MB  | nb 5 (reranker) |

### Datasets

| dataset                                              | how it's used                                | used in |
|------------------------------------------------------|----------------------------------------------|---------|
| [`sentence-transformers/simple-wiki`](https://huggingface.co/datasets/sentence-transformers/simple-wiki) | sentence pairs; grouped into wiki passages   | nb 3, 5, 8 |

Datasets are streamed (`streaming=True`) and only the first ~150–500
sentences are pulled — total transfer is a few hundred KB per notebook,
not the whole corpus.

### Gemini API

The `google-genai` Python SDK calls `gemini-3-flash-preview` for generation
(notebooks 3, 6, 7, 8). The free tier of AI Studio is enough to complete
the entire tutorial; the per-notebook cost is a handful of short prompts.
If the model name changes, edit `GEN_MODEL` / `JUDGE_MODEL` in the relevant
cell of each notebook.

**Rate limit behavior.** The notebooks deliberately do **not** retry on
429 / 503. If a Gemini call fails, the traceback appears immediately —
wait ~30 seconds and re-run the failing cell. (Silent retries with long
sleeps make notebooks look hung; we'd rather fail loudly.) Per-day free-tier
quotas can be exhausted — if so, either swap the model name to one that
still has quota or wait until midnight Pacific for the quota to reset.

## Working with HuggingFace rate limits (optional)

The HF Hub rate-limits anonymous downloads. If you hit a 429, set a token:

```powershell
$env:HF_TOKEN = "hf_...your-token..."
```

Get a free read-only token from <https://huggingface.co/settings/tokens>.

## Regenerating notebooks

The `.ipynb` files in `tutorial/` are emitted by the scripts in `scripts/`.
If you want to tweak content, edit the script and rebuild:

```powershell
python scripts\build_03_basic_rag.py
```

Then execute the notebook end-to-end to populate outputs:

```powershell
python -m jupyter nbconvert --to notebook --execute --inplace `
    tutorial\03_basic_rag.ipynb `
    --ExecutePreprocessor.timeout=600
```

## Troubleshooting

- **`ModuleNotFoundError` inside a notebook** — the kernel is not pointing
  at the venv. Reselect the `RAG tutorial (venv)` kernel.
- **`Failed to connect to huggingface.co`** — first run needs internet for
  the model / dataset download. After that everything runs offline.
- **`google.genai.errors.ClientError: 400 INVALID_ARGUMENT`** — `key.env`
  is missing, empty, or contains extra characters (a trailing newline or
  `KEY=` prefix). Verify with `cat key.env` / `Get-Content key.env -Raw`.
- **`torch` install fails on Windows** — make sure your Python is 64-bit
  and not the Microsoft Store stub (`where.exe python`). The dev/test
  machine for this repo uses `C:\Python314\python.exe`.
